Build a complete **Student Portal** for LearnSync using **React (functional components + hooks)** and **Tailwind CSS**.

The Student Portal must act as:
👉 A **personal tutor + planner + progress tracker**

All data must be handled using **localStorage (mock data)**.

The UI must be **clean, minimal, professional**, with **NO gradients**.

---

# 🧠 CORE OBJECTIVE

The Student Portal must:

* Personalize learning instantly
* Adapt UI based on age
* Provide structured course learning
* Guide daily study using a planner
* Track progress & performance
* Allow collaborative doubt resolution
* Use AI for assistance

# 🤖 AI-POWERED LEARNING PIPELINE (STUDENT SIDE — CRITICAL FEATURE)

Implement a system where mentor-provided content is transformed into personalized learning modules for students using AI (Gemini 2.5 Flash).

---

# 🧑‍🏫 STEP 1: MENTOR PROVIDES MATERIAL

Mentor uploads or inputs:

* Topic (e.g., “Fractions Basics”)
* Content (text / notes)

Store in:

```js
{
  topic: "Fractions",
  content: "definition and examples...",
  difficulty: "basic"
}
```

---

# 🤖 STEP 2: AI CONTENT PROCESSING

When student opens the topic:

Call Gemini with:

Prompt:

"Convert the following content into a structured learning module for a {student_level} student. Include:

1. Simple explanation
2. Key points
3. Examples
4. Summary"

---

# 📘 STEP 3: STUDENT LEARNING MODULE (UI)

Render as:

### Section 1: Simplified Explanation

* Easy-to-read content

---

### Section 2: Key Points

* Bullet list

---

### Section 3: Examples

* Simple examples

---

### Section 4: Quick Summary

* Short recap

---

---

# 🧪 STEP 4: AI QUIZ GENERATION

Use student data:

* Level
* Weak topics
* Past performance

---

Prompt:

"Generate 5 questions for the topic 'Fractions' for a {student_level} student. Focus more on weak areas."

---

Output:

* 5 questions
* Difficulty tags

---

---

# 🎯 STEP 5: PERSONALIZATION (VERY IMPORTANT)

Use:

```js
studentData = {
  level: "beginner",
  weakTopics: ["fractions"]
}
```

---

Rules:

* Weak topics → more questions
* Beginner → simple language
* Strong topics → fewer questions

---

---

# 📊 STEP 6: AFTER STUDENT COMPLETES QUIZ

Store:

```js
{
  score: 3,
  topic: "Fractions"
}
```

---

Call Gemini:

Prompt:

"A student scored 3/5 in Fractions. Suggest what they should revise next."

---

Output:

* Feedback
* Next topic suggestion

---

---

# 🔄 STEP 7: SYSTEM UPDATES

Update:

* Weak topics
* Progress
* Study plan
